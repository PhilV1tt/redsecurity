\subhead{Where the project stands}
BeSecured is on schedule. The PowerShell prototype has been rebuilt into a cross-platform Python scanner with a local visual interface. Weeks 1 to 7 are complete: the scanner runs on Windows, macOS and Linux, computes an explainable 0 to 100 risk score, and reports findings by severity with plain-language fixes. The interface now opens in its own window and reads like a system diagnostic tool rather than a web page. Week 8 covers packaging and quality work. The whole tool runs locally, with no account, no backend and no data upload, and a test suite of 59 checks guards the scan contract and the privacy promise.

\vspace{4pt}
\begin{callout}
\textbf{What changed since the v2 report.} The earlier report described an intended React / Tailwind / Chart.js stack. The team chose a lighter path that fits a privacy-first local tool: a Python package with no third-party dependencies and a hand-built local web UI. This report reflects what was actually built.
\end{callout}

% ===== PAGE 4: ARCHITECTURE =====
\sechead{3. Architecture and what was built}{Cross-platform Python scanner with a local web interface}

A single entry point runs the common checks, then dispatches to the checks for the current operating system, aggregates the findings, and computes the score. The interface is served by a small local Python server and talks to the scanner through one local endpoint.

\vspace{2pt}
\begin{tcolorbox}[colback=rowtint,colframe=linesoft,boxrule=0.4pt,arc=1.5pt,left=8pt,right=8pt,top=5pt,bottom=5pt]
\ttfamily\small Local UI or CLI $\rightarrow$ Python scanner $\rightarrow$ OS-specific checks $\rightarrow$ risk scoring $\rightarrow$ HTML or JSON report
\end{tcolorbox}

\subhead{Module map}
\rowcolors{2}{white}{rowtint}
\begin{tabularx}{\linewidth}{@{}>{\ttfamily\bfseries}p{34mm}X@{}}
  \rowcolor{accentsoft} \textbf{\normalfont Module} & \textbf{Responsibility} \\
  scanner.py & Entry point. Runs common checks, dispatches to the OS module, aggregates findings, builds the result. \\
  checks/common.py & System info, privilege detection, listening ports, cross-platform helpers. \\
  checks/windows.py & Firewall, Defender, updates, users, password policy, shares, startup, UAC, BitLocker. \\
  checks/macos.py & Firewall, FileVault, Gatekeeper, SIP, XProtect, users, sharing, launch items. \\
  checks/linux.py & Firewall, updates, LUKS, users, password rules, shares, startup, antivirus. \\
  models.py & Finding and ScanResult data contracts, with a stable JSON schema. \\
  scoring.py & Score, grade, per-category and per-finding impact, calculation steps. \\
  report.py & HTML and JSON report rendering. \\
  ui/ & Local server and browser interface. Opens in a dedicated app window. \\
\end{tabularx}

\subhead{What gets checked}
\rowcolors{2}{white}{rowtint}
\begin{tabularx}{\linewidth}{@{}>{\bfseries}p{30mm}Xl@{}}

  Protection & Antivirus or native protection and signature freshness & All \\
  Disk encryption & BitLocker, FileVault or LUKS state & All \\
  Hardening & UAC on Windows, Gatekeeper and SIP on macOS & Win / macOS \\
\end{tabularx}

\sechead{4. Scoring, privacy and quality}{An explainable score, a local-first design, and an automated test suite}

\subhead{Scoring model}
\begin{minipage}[t]{0.40\linewidth}
\rowcolors{2}{white}{rowtint}
\begin{tabularx}{\linewidth}{@{}Xr@{}}
  \rowcolor{accentsoft} \textbf{Status} & \textbf{Points lost} \\
  \tg{ok}{OK} & 0 \\
  \tg{warn}{WARN} & 2 \\
  \tg{crit}{CRIT} & 5 \\
  INFO & ignored \\
  SKIP & ignored \\
\end{tabularx}
\end{minipage}\hfill
\begin{minipage}[t]{0.56\linewidth}
Only OK, WARN and CRIT change the score. INFO and SKIP are shown for transparency but never move the number, so an unavailable check is never counted as a pass.
\vspace{4pt}
\begin{tcolorbox}[colback=rowtint,colframe=linesoft,boxrule=0.4pt,arc=1.5pt,left=8pt,right=8pt,top=4pt,bottom=4pt]
\ttfamily\small score = 100 - round(lost / max $\times$ 100)
\end{tcolorbox}
Grades: A (90-100), B (75-89), C (60-74), D (40-59), F (0-39).
\end{minipage}

\vspace{8pt}
\begin{minipage}[t]{0.47\linewidth}
\subhead{Privacy guarantees}
\begin{itemize}[leftmargin=13pt,itemsep=2pt,topsep=1pt]
  \item No account, no remote backend, no cloud API.
  \item No upload of files, credentials or system details.
  \item The UI server binds to 127.0.0.1 and exposes only a local scan endpoint.
  \item Reports are written to a local app-data folder.
  \item The CLI refuses to write a report to a cloud-synced or remote path.
  \item A test asserts the interface assets contain no remote URLs.
\end{itemize}
\end{minipage}\hfill
\begin{minipage}[t]{0.47\linewidth}
\subhead{Test suite (59 checks)}
\begin{itemize}[leftmargin=13pt,itemsep=2pt,topsep=1pt]
  \item Windows, macOS and Linux check parsers.
  \item Network and port parsing.
  \item Scoring, grades and calculation steps.
  \item HTML and JSON report export.
  \item UI JSON contract and finding shape.
  \item Local \code{/api/scan} endpoint.
  \item No remote-reference guard over the sensitive sources.
\end{itemize}
\end{minipage}
