## 🎯 1. Project Objectives
The objective of this project is to design and develop a user-friendly cybersecurity risk scanner tailored for individuals and small organizations with limited technical expertise. The system will aim to:

*   Provide a simple and automated way to assess a user’s cybersecurity posture.
*   Identify common security weaknesses through safe and non-intrusive system checks.
*   Translate technical findings into clear, understandable insights for non-expert users.
*   Generate a reliable and interpretable cybersecurity risk score based on detected issues.
*   Offer practical, prioritized recommendations that users can easily implement.
*   Ensure the solution remains lightweight, accessible, and respectful of user privacy and system limitations.

The project focuses not only on detection, but also on awareness and education, helping users understand their risks and take meaningful actions to improve their security.

## 💼 2. Business Need and Benefits
This project addresses the need for an accessible cybersecurity tool for non-experts.
**Benefits include:**
*   Increased cybersecurity awareness.
*   Reduced exposure to common attacks.
*   Affordable and simple risk assessment.
*   Improved user security practices.
*   Potential development into a SaaS product.

## 📏 3. Project Scope

### ✅ In Scope
The project will:
1.  Run basic and non-intrusive checks on the user's environment when launched.
2.  Quietly analyze collected information to evaluate the overall security posture.
3.  Translate technical findings into a simple cybersecurity risk score.
4.  Present an intuitive dashboard highlighting key issues with actionable, step-by-step recommendations.

**Intentional Exclusion:** The project avoids complex features like advanced penetration testing or real-time monitoring to maintain focus on clarity and usability for non-experts.

### 🚫 Out of Scope
*   Advanced penetration testing techniques (e.g., exploitation, privilege escalation).
*   Continuous or real-time monitoring of user systems.
*   Enterprise-grade security management features (e.g., SIEM integration).
*   Automated threat response that modifies system configurations without explicit user approval.
*   Deep network scanning beyond basic and safe checks.
*   Storage or transmission of sensitive personal data outside the local environment.

### ✅ 6. Success Criteria
The project is successful if the following criteria are met:
*   The application executes all core features without critical errors in at least **95%** of test runs.
*   The system successfully generates a cybersecurity risk score for **100%** of valid scans.
*   At least **80%** of identified vulnerabilities are correctly detected in predefined test scenarios.
*   Recommendations are rated as "clear and useful" by at least **80%** of test users.
*   The user interface allows a new user to complete a scan without assistance in under **5 minutes**.
